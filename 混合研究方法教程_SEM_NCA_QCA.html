教程概述

为什么需要混合研究方法？

传统的研究方法往往存在以下局限：

  1. 因果逻辑单一：大多数研究只关注”充分性”（X对Y的影响），忽视了”必要性”（X是Y产生的前提）和”组态性”（多个X组合导致Y）

  2. 方法局限性：

    • 回归/SEM：只能捕捉平均净效应，假设因果对称性
    • 定性研究：擅长深度理解，但难以量化效应大小
    • 单一方法：无法全面捕捉因果关系的复杂性

  3. 现实世界的因果复杂性：社会现象的原因往往是”多重并发”的——多个条件以组合方式共同导致结果，而非各自独立起作用

本教程的核心价值

本教程将教你如何整合三种互补的方法：

方法 回答的问题 因果逻辑 你的收获
SEM (结构方程模型) X对Y的”平均效应”有多大？ 充分性（对称） 量化变量间的净效应和中介机制
NCA (必要条件分析) X是Y的”必要条件”吗？ 必要性（非对称） 识别瓶颈条件和约束因素
QCA (定性比较分析) 哪些条件组合导致Y？ 充分性（组态、非对称） 发现多重并发因果路径和等效性

理论基础与方法论原则

核心方法论原则

1. 因果多元主义

核心观点：因果关系存在多种形态，没有单一的最佳分析方法。

  • 充分性（Sufficiency）：X的出现导致Y的出现（传统回归关注的问题）
  • 必要性（Necessity）：Y的出现必须以X为前提（NCA关注的问题）
  • 组态性（Configurationality）：多个X以特定组合方式导致Y（QCA关注的问题）

实践含义：根据你的研究问题，选择最适配的因果逻辑和方法。

2. 方法-问题适配

核心观点：研究方法必须由研究问题决定，而非相反。

决策框架：

研究问题 → 因果逻辑 → 方法选择
  ↓
"X对Y有什么影响？" → 充分性 → SEM
"什么是Y的必要条件？" → 必要性 → NCA
"哪些因素组合导致Y？" → 组态性 → QCA
"如何全面理解Y的因果关系？" → 多种逻辑 → 混合方法

3. 互补性思维

核心观点：不同方法识别因果关系的不同维度，联合使用提供更全面理解。

三种方法的互补性： - SEM揭示”平均效应” → 告诉你”一般情况下X对Y的影响” - NCA识别”必要条件” → 告诉你”什么是Y产生不可或缺的” - QCA探索”组态路径” → 告诉你”哪些条件组合可以导致Y”

4. 理论驱动原则

核心观点：方法选择和使用必须理论驱动。

关键要求： - 使用NCA之前，必须理论论证为什么X可能是Y的必要条件 - 使用QCA之前，必须理论论证为什么因果可能是组态的 - 不能”先跑一下看看结果”再找理论解释

5. 因果复杂性认可

核心观点：社会现象原因是”多重并发”的，等效性（equifinality）是常态。

核心概念： - 等效性：达到同一结果可以有多种不同的路径 - 非对称性：导致Y出现的因素组合，与导致Y缺失的因素组合不一定完全对称 - 组态效应：整体大于部分之和，条件的组合效应不等于各条件独立效应之和

决策树2：混合方法设计选择

混合方法设计
  ↓
研究阶段？
  ├─ 探索性研究 → 顺序1: QCA → SEM → NCA
  ├─ 验证性研究 → 顺序2: SEM → NCA → QCA  
  └─ 综合性研究 → 顺序3: NCA → QCA → SEM

三种顺序的适用场景：

  1. 顺序1（探索→验证→深化）：
    • 适用：理论不足，需要先发现条件组合
    • 优点：从探索性发现到验证性检验，逻辑清晰
    • 缺点：需要较大样本量支持SEM
  2. 顺序2（验证→检验→探索）：
    • 适用：理论充分，需要检验后探索未预期路径
    • 优点：先验证理论模型，再用NCA和QCA深化理解
    • 缺点：如果SEM结果不显著，后续分析可能缺乏理论支撑
  3. 顺序3（约束→路径→量化）：
    • 适用：需要先识别瓶颈，再探索突破约束的路径
    • 优点：从必要条件（约束）到组态路径（突破），逻辑连贯
    • 缺点：NCA的理论论证要求较高

三种方法的功能分工

SEM（结构方程模型）

主要功能

  1. 检验理论模型：验证潜变量间的假设关系
  2. 量化效应大小：提供标准化系数、效应量
  3. 检验中介效应：分析变量间的作用机制
  4. 处理测量误差：通过CFA确保构念测量的信效度

适用场景

  • ✅ 检验理论模型中的直接效应和中介效应
  • ✅ 需要量化效应大小和统计显著性
  • ✅ 关注”平均效应”（average effect）
  • ❌ 不适合识别必要条件
  • ❌ 不适合探索组态效应

方法变体

  1. cb-SEM（协方差基础SEM）：
    • 软件：AMOS, Mplus, lavaan (R)
    • 要求：大样本（N > 200），数据正态分布
    • 优点：参数估计准确，模型检验严格
    • 缺点：对数据要求高
  2. PLS-SEM（偏最小二乘SEM）：
    • 软件：SmartPLS, plspm (R)
    • 要求：小样本可接受（N > 30），不要求正态分布
    • 优点：对数据要求宽松，适合探索性研究
    • 缺点：参数估计可能有偏，不适合严格的理论验证

结果解读

  • 拟合指数：CFI > 0.95, TLI > 0.95, RMSEA < 0.06, SRMR < 0.08
  • 路径系数：标准化系数（β），通常0.1-0.3为小效应，0.3-0.5为中等效应，>0.5为大效应
  • 显著性：p < 0.05（或使用Bootstrap法）
  • 效应量：Cohen’s f²（0.02为小，0.15为中等，0.35为大）

NCA（必要条件分析）

主要功能

  1. 识别必要条件：判断X是否是Y产生不可或缺的条件
  2. 量化约束程度：通过效应量（d值）反映必要条件的约束程度
  3. 瓶颈分析：揭示达成不同水平的Y所需的X的最低必要值

适用场景

  • ✅ 理论论证了某个条件是结果产生的必要条件
  • ✅ 关注”瓶颈因素”和”约束条件”
  • ✅ 需要区分”应该有的条件”（should-have）和”必须有的条件”（must-have）
  • ❌ 不适合识别充分条件（X对Y的平均效应）
  • ❌ 不适合探索多个条件的组合效应

理论基础

必要条件的逻辑： - 如果X是Y的必要条件，则：Y → X（Y出现必须以X为前提） - 用集合语言表达：Y的集合是X的集合的子集 - 用逻辑表达：~X → ~Y（X不存在则Y不可能出现）

与充分条件的区别： - 充分条件：X → Y（X出现导致Y出现） - 必要条件：Y → X（Y出现必须以X为前提） - 一个条件可以既是必要也是充分，或仅是必要的，或仅充分的

结果解读

  • 效应量（d值）：

    • d = 0.0-0.1：可忽略的必要性
    • d = 0.1-0.3：小效应
    • d = 0.3-0.5：中等效应
    • d > 0.5：大效应

  • 显著性检验：置换检验（Permutation Test）1000次，p < 0.05表示必要条件关系显著

  • 天花板线图：可视化X和Y的关系，展示”天花板”（上限）

  • 瓶颈表：显示达成不同水平的Y所需的X的最低必要值

关键要求

⚠️ 理论论证优先：使用NCA之前，必须理论论证为什么X可能是Y的必要条件。不能”先跑一下看看”。

⚠️ 时间顺序：X的发生必须早于Y，或至少在逻辑上先于Y。

⚠️ 因果机制：必须能够解释X如何影响Y的产生。

QCA（定性比较分析）

主要功能

  1. 识别组态路径：发现导致结果的多个条件组合
  2. 揭示等效性：展示达到同一结果的不同路径
  3. 探索因果不对称性：分别分析导致结果出现和缺失的条件组合
  4. 处理中小样本：适合案例数量在10-150的研究

适用场景

  • ✅ 理论论证了因果的组态性（多个条件组合导致结果）
  • ✅ 关注”多重并发因果”
  • ✅ 样本量中等（10-150案例）
  • ✅ 需要揭示因果不对称性
  • ❌ 不适合大样本（>150案例，除非使用扩展方法）
  • ❌ 不适合检验简单的线性关系

方法变体

  1. csQCA（清晰集QCA）：
    • 变量：二分类（0/1）
    • 适用：案例可以清晰归类为”属于”或”不属于”
    • 优点：简单直观
    • 缺点：损失信息，不适合连续变量
  2. fsQCA（模糊集QCA）：
    • 变量：模糊集（0-1之间的连续值）
    • 适用：案例可以部分属于某个集合
    • 优点：保留更多信息，适合连续变量
    • 缺点：校准过程主观，需要理论依据
  3. mvQCA（多值QCA）：
    • 变量：多分类（0/1/2…）
    • 适用：分类变量有多于两个类别
    • 优点：适合有序分类变量
    • 缺点：解释相对复杂

核心概念

  1. 校准（Calibration）：
    • 将原始数据转换为集合隶属分数（0-1）
    • 三个定性锚点：完全不属于（e）、交叉点（c）、完全属于（i）
    • 校准方案必须有理论或数据分布依据
  2. 一致性（Consistency）：
    • 衡量条件组合导致结果的程度
    • 必要条件：一致性 ≥ 0.9
    • 充分性：一致性 ≥ 0.75（探索性）或0.8（验证性）
  3. 覆盖度（Coverage）：
    • 衡量条件组合解释案例的比例
    • 原始覆盖度：所有案例中被解释的
    • 唯一覆盖度：排除与其他解重叠后被唯一解释的
  4. 组态解：
    • 简约解（Parsimonious solution）：最简洁的组态
    • 中间解（Intermediate solution）：加入理论知识的中间解
    • 复杂解（Complex solution）：包含所有可能的解

结果解读

  • 必要性分析：一致性 ≥ 0.9 → 该条件是必要条件
  • 充分性分析：组态一致性 ≥ 0.75，覆盖度尽可能高
  • 核心条件：在简约解和中间解中都出现的条件
  • 边缘条件：仅在中间解中出现的条件

研究设计阶段

阶段1：明确研究问题

🔴 CHECKPOINT · 研究问题明确性检查

在开始之前，必须明确：

  1. 写下一句话研究问题：用一句话概括你的研究问题
  2. 识别子问题：将主问题分解为3-5个子问题
  3. 匹配因果逻辑：每个子问题对应哪种因果逻辑（充分性/必要性/组态性）？
  4. 选择方法：基于子问题的因果逻辑，选择最适配的方法

示例：

主问题：“哪些因素导致员工创新行为？”

子问题： 1. “领导风格对员工创新行为有什么影响？” → 充分性 → SEM 2. “创新自我效能感是否是员工创新行为的必要条件？” → 必要性 → NCA 3. “哪些因素组合可以导致员工创新行为？” → 组态性 → QCA

方法选择：混合方法（SEM + NCA + QCA）

决策规则：

  • 如果研究问题明确是”X对Y的影响” → 优先SEM
  • 如果研究问题涉及”必要条件” → 必须包含NCA
  • 如果研究问题涉及”多重并发因果” → 必须包含QCA
  • 如果理论支持多种因果逻辑 → 使用混合方法

🛑 STOP：请确认研究问题和理论匹配是否已经明确？如果不确定，必须先回到文献回顾阶段明确研究问题后再继续。

阶段2：评估理论匹配

核心任务：回顾文献，识别理论假设的因果逻辑。

步骤：

  1. 文献回顾：系统回顾与研究问题相关的前人研究
  2. 理论识别：识别前人研究使用的理论框架
  3. 因果逻辑提取：从理论框架中提取隐含的因果逻辑
  4. 缺口识别：识别前人研究忽略的因果逻辑

示例：

理论框架：社会认知理论（Bandura）

  • 隐含的充分性逻辑：自我效能感 → 创新行为（主效应）
  • 隐含的必要性逻辑：自我效能感可能是创新行为的必要条件（“没有自我效能感，创新行为不可能发生”）
  • 忽略的组态逻辑：自我效能感、领导支持、组织氛围的组合效应

理论匹配评估：

  • ✅ 充分性逻辑有明确理论支持 → 使用SEM
  • ✅ 必要性逻辑可以理论论证 → 使用NCA
  • ✅ 组态逻辑被前人忽略 → 使用QCA探索

结论：使用混合方法（SEM + NCA + QCA）

阶段3：设计分析顺序

核心任务：确定方法的执行顺序和整合策略。

三种顺序的详细对比：

顺序 分析流程 适用场景 优点 缺点
顺序1 QCA → SEM → NCA 探索性研究 从探索性发现到验证性检验 需要较大样本量
顺序2 SEM → NCA → QCA 验证性研究 先验证理论模型 如果SEM不显著，后续分析缺乏支撑
顺序3 NCA → QCA → SEM 综合性研究 从约束到突破路径 NCA的理论论证要求高

选择指南：

  1. 如果你处于探索性阶段（理论不足，样本量中等）：
    • 推荐：顺序1（QCA → SEM → NCA）
    • 理由：先用QCA探索条件组合，再用SEM验证量化效应，最后用NCA识别瓶颈
  2. 如果你处于验证性阶段（理论充分，样本量充足）：
    • 推荐：顺序2（SEM → NCA → QCA）
    • 理由：先验证理论模型，再用NCA检验必要条件，最后用QCA探索未预期的组态路径
  3. 如果你关注瓶颈突破（理论关注约束条件）：
    • 推荐：顺序3（NCA → QCA → SEM）
    • 理由：先识别瓶颈（必要条件），再探索突破瓶颈的路径（组态），最后量化效应大小

阶段4：确定整合策略

核心任务：设计如何整合三种方法的结果。

整合策略1：分别报告，然后整合（推荐）

步骤： 1. 按选择的分析顺序，分别报告三种方法的结果 2. 识别三种方法结果的一致点 3. 解释结果之间的矛盾（使用因果不对称性） 4. 提供整合性理论框架

优点： - 清晰展示每种方法的价值 - 便于读者理解 - 符合顶刊报告规范

缺点： - 篇幅较长 - 需要更多的整合解释工作

整合策略2：按研究问题整合

步骤： 1. 将研究问题分解为3-5个子问题 2. 每个子问题下，分别报告三种方法的发现 3. 每个子问题提供整合解释 4. 最后提供总体整合框架

优点： - 紧扣研究问题 - 逻辑清晰 - 便于突出理论贡献

缺点： - 需要在研究设计阶段就明确子问题 - 对研究问题的分解要求较高

推荐：使用”整合策略1”（分别报告，然后整合），除非你的研究问题非常清晰且可以自然分解。

测量工具选择

核心要求：必须使用已验证的量表（Validated Scales）。

步骤：

  1. 文献回顾：找到与研究构念相关的前人研究
  2. 量表识别：识别前人研究使用的量表
  3. 信效度检查：检查量表的信度（α > 0.7）和效度（CFA拟合指数达标）
  4. 跨文化适配：如果用到不同文化语境，需要进行翻译和跨文化验证

量表选择标准：

  • ✅ 已发表在国际期刊上的量表
  • ✅ 有证据表明信效度良好
  • ✅ 被用于类似的研究情境
  • ❌ 自编量表（除非经过严格的量表开发程序）
  • ❌ 信效度证据不足的量表

数据收集注意事项

多时点设计：

如果研究关注的是因果关系（而非仅相关关系），推荐使用多时点数据收集：

  • 时点1：收集自变量、控制变量
  • 时点2（间隔2-4周）：收集中介变量
  • 时点3（间隔2-4周）：收集因变量

优点： - 减少共同方法偏差（CMV） - 增强因果推断的逻辑 - 符合顶刊的方法论要求

缺点： - 样本流失率可能较高 - 需要更多的资源和时间

共同方法偏差（CMV）检验：

Harman单因子检验： - 将所有题项放入探索性因子分析 - 如果第一个因子解释的变异量 < 50%，则CMV不严重

控制变量：

在收集数据时，考虑收集以下控制变量： - 人口统计学变量：年龄、性别、教育水平、工作年限 - 组织变量：组织规模、行业、所有权性质 - 其他相关变量：基于文献回顾

  Latent2 =~ y1 + y2 + y3

QCA分析实战

阶段1：数据校准（Calibration）

核心任务：将原始数据转换为模糊集隶属分数（0-1）。

校准原则：

  1. 理论依据：校准锚点必须有理论依据
  2. 三个定性锚点：
    • e：完全不属于（完全不隶属该集合）
    • c：交叉点（既不完全属于也不完全不属于）
    • i：完全属于（完全隶属该集合）
  3. 锚点选择：
    • 基于理论（如”高收入”的理论定义）
    • 基于数据分布（95th/50th/5th 分位数）
    • 报告校准合理性检验

步骤1：选择校准方案

方案1：理论锚点法 - 适用于：理论明确界定了”高/中/低”的标准 - 示例：如果理论定义”高创新自我效能感”为”得分≥4（5点量表）“，则i=4

方案2：分位数法 - 适用于：理论未明确界定锚点 - 示例：使用95th/50th/5th分位数作为i/c/e

完整案例分析

案例：员工创新行为的影响因素研究

研究问题：“哪些因素导致员工创新行为？”

理论框架：社会认知理论 + 组织支持理论

研究方法：SEM + NCA + QCA混合方法

样本：N = 250名员工（来自10家中国企业）

变量： - 自变量：领导支持、同事支持、自我效能感、组织创新氛围 - 因变量：创新行为

分析步骤

步骤1：SEM分析

测量模型： - 领导支持：4个题项，CFA拟合指数达标 - 同事支持：4个题项，CFA拟合指数达标 - 自我效能感：3个题项，CFA拟合指数达标 - 组织创新氛围：4个题项，CFA拟合指数达标 - 创新行为：4个题项，CFA拟合指数达标

结构模型： - 领导支持 → 创新行为：β = 0.25, p < 0.01 - 同事支持 → 创新行为：β = 0.18, p < 0.05 - 自我效能感 → 创新行为：β = 0.32, p < 0.001 - 组织创新氛围 → 创新行为：β = 0.15, p < 0.05

SEM结论：所有四个前因变量对创新行为有显著正向影响。

步骤2：NCA分析

理论论证：基于社会认知理论，自我效能感是创新行为产生的必要条件（“没有自我效能感，员工不会尝试创新”）。

NCA结果： - 自我效能感是创新行为的必要条件：d = 0.35, p < 0.01 - 其他条件不是必要条件：d < 0.1, p > 0.05

瓶颈分析： - 要达到50%的创新行为，至少需要30%的自我效能感 - 要达到100%的创新行为，至少需要70%的自我效能感

NCA结论：自我效能感是创新行为的必要条件，但其他条件不是。

步骤3：QCA分析

校准方案：使用理论锚点法 + 分位数法，进行敏感性检验。

必要性分析： - 自我效能感的一致性 = 0.92 → 必要条件 - 其他条件的一致性 < 0.9 → 不是必要条件

充分性分析： - 识别出3条导致高创新行为的组态（一致性≥0.85，覆盖度=0.68）： - 组态1：高领导支持 * 高自我效能感（核心条件：领导支持、自我效能感） - 组态2：高同事支持 * 高自我效能感（核心条件：同事支持、自我效能感） - 组态3：高组织创新氛围 * 高工作自主性（核心条件：组织创新氛围、工作自主性；自我效能感为边缘条件）

QCA结论：自我效能感出现在所有组态中，表明其重要性。但创新行为可以通过多种路径产生（等效性）。

步骤4：结果整合

一致点： - SEM和QCA都显示自我效能感对创新行为有积极影响 - NCA和QCA都显示自我效能感是必要条件

矛盾点： - SEM显示领导支持对创新行为有显著正向影响，但QCA显示领导支持仅出现在部分组态中（非普遍存在）

解释： - SEM关注的是”平均效应”，即一般情况下领导支持对创新行为有正向影响 - QCA关注的是”组态效应”，即领导支持需要与其他条件组合才能导致创新行为 - 这一”矛盾”揭示了因果不对称性：领导支持的平均效应显著，但不是所有导致创新行为的组态都包含领导支持

整合理论： - 平均效应维度：领导支持、同事支持、自我效能感、组织创新氛围都对创新行为有正向影响（SEM） - 必要条件维度：自我效能感是创新行为的必要条件，构成瓶颈（NCA） - 组态路径维度：创新行为可以通过多种路径产生，但所有路径都包含自我效能感（QCA）

理论贡献： 1. 揭示了自我效能感作为必要条件的”瓶颈效应”，超越了前人仅关注充分性的研究 2. 发现了导致创新行为的多重路径，揭示了因果复杂性和等效性 3. 整合了充分性和必要性的因果逻辑，提供了更全面的因果理解

最佳实践与常见误区

最佳实践

1. 理论驱动

原则：方法选择和使用必须理论驱动。

实践： - 使用NCA之前，必须理论论证为什么X可能是Y的必要条件 - 使用QCA之前，必须理论论证为什么因果可能是组态的 - 不能”先跑一下看看结果”再找理论解释

2. 方法适配

原则：研究方法必须由研究问题决定。

实践： - 明确研究问题的因果逻辑（充分性/必要性/组态性） - 选择最适配的方法或方法组合 - 避免方法堆砌（不能使用三种方法”因为看起来厉害”）

3. 结果整合

原则：混合方法的价值在于整合，而非简单叠加。

实践： - 分别报告每种方法的结果 - 识别结果之间的一致点和矛盾点 - 提供整合性理论框架 - 解释看似矛盾的结果（使用因果不对称性）

4. 透明报告

原则：报告必须透明、可复制。

实践： - 详细报告校准方案（QCA） - 详细报告效应量和显著性检验（NCA） - 详细报告拟合指数和路径系数（SEM） - 提供稳健性检验结果

5. 局限坦诚

原则：坦诚说明研究的局限性。

实践： - 说明横截面设计的局限（如果不能做纵向设计） - 说明共同方法偏差的潜在影响 - 说明样本代表性的局限 - 说明方法的固有局限（如SEM的对称性假设）

常见误区

误区1：理论未明确必要性，但使用了NCA

表现：在方法章节写”我们使用NCA探索必要条件”，但没有理论论证。

后果：审稿人会质疑”你凭什么说这个条件是必要条件？”

纠正：先发展必要条件理论，再使用NCA。如果理论未明确，改用SEM主效应分析。

误区2：忽略QCA的因果不对称性

表现：只分析导致结果出现的组态，不分析导致结果缺失的组态。

后果：遗漏了重要的因果信息，结论可能偏颇。

纠正：分别分析导致结果出现和缺失的组态，比较异同。

误区3：将NCA的效应量解释为影响力大小

表现：在讨论中写”NCA显示X的效应量很大（d=0.6），说明X对Y的影响很大”。

后果：误解了NCA的效应量含义。

纠正：NCA的效应量（d值）仅表示必要条件的约束程度，而非影响力大小。应该解释为”X是Y产生的瓶颈条件，约束程度为d=0.6”。

误区4：混合方法但未整合结果

表现：分别报告了三种方法的结果，但没有提供整合解释。

后果：混合方法的价值未体现，看起来像”方法堆砌”。

纠正：使用整合解释框架，提供综合性结论。可以分别报告，然后整合解释。

误区5：校准锚点选择缺乏理论依据

表现：在QCA校准时，随意选择锚点，或仅基于数据分布选择。

后果：校准结果不可靠，QCA结果缺乏理论支撑。

纠正：校准锚点必须有理论或分位数依据。如果理论未明确，使用分位数法，并报告敏感性检验。

误区6：样本量不足但想用混合方法

表现：样本量 < 200，但想使用SEM + NCA + QCA混合方法。

后果：SEM的cb-SEM需要大样本，结果可能不可靠。

纠正：增加样本量至>200，或改为单一方法（优先SEM或fsQCA）。

  Latent3 ~ Latent1 + Latent2
'

# 查看SEM结果

  # 测量模型
  Latent1 =~ x1 + x2 + x3
  Latent2 =~ m1 + m2 + m3
  Latent3 =~ y1 + y2 + y3

  Latent3 ~ b*Latent2 + c*Latent1

总结

本教程系统介绍了如何整合SEM、NCA和QCA三种方法进行混合研究。核心要点：

  1. 理论驱动：方法选择和使用必须理论驱动，不能”先跑一下看看”
  2. 方法适配：根据研究问题的因果逻辑选择最适配的方法
  3. 互补整合：三种方法各擅其长，互补而非替代
  4. 结果整合：混合方法的价值在于整合，需提供综合性理论框架
  5. 透明报告：详细报告分析步骤和结果，便于复制和验证

三种方法的整合价值：

  • SEM揭示了”平均效应” → 告诉你”一般情况下X对Y的影响”
  • NCA识别了”必要条件” → 告诉你”什么是Y产生不可或缺的”
  • QCA探索了”组态路径” → 告诉你”哪些条件组合可以导致Y”

三种方法的结合，能够提供更全面、更深入的因果理解，超越单一方法的局限。
